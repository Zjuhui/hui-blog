# v0.2 计划

v0.2 的目标不是增加很多功能，而是把博客从“能打开的首页”推进到“可以长期写作的系统”。

## 当前状态

- 已迁移到 Astro。
- 首页完成 Apple-like 毛玻璃视觉方向。
- GitHub Pages 可访问。
- Notion 工作台已建立。
- 首页学习笔记已建立。

## v0.2 核心目标

1. 文章从手写页面迁移到 Markdown 内容集合。
2. 自动生成文章列表和文章详情页。
3. 完成 Research / Life / Works / Archive 的基础页面。
4. 建立文章 frontmatter 规范。
5. 继续记录 token ledger。

## 任务拆分

### 1. 内容系统

新增：

```text
src/content/blog/
```

每篇文章用 Markdown 写：

```markdown
---
title: 为什么要做这个博客
date: 2026-06-08
section: life
language: zh-en
status: draft
---

# 正文
```

这样 Astro 可以自动读取标题、日期、分类和正文。

### 2. 文章列表

把 `WeekOnePosts.astro` 从“写死两篇文章”改成“读取 Markdown 文章”。

逻辑会变成：

```text
读取 src/content/blog/*.md
  -> 按日期排序
  -> 渲染文章卡片
```

### 3. 文章详情页

新增动态路由：

```text
src/pages/posts/[slug].astro
```

这个文件负责根据文章 slug 自动生成每篇文章的网页。

### 4. 分类页

Research / Life / Works 不再只是占位页，而是显示对应 section 的文章。

例如：

```text
/research/ 只显示 section: research 的文章
/life/      只显示 section: life 的文章
/works/     只显示 section: works 的作品记录
```

### 5. 视觉改进

保持 v0.2 的视觉原则：

- 浅色背景
- 毛玻璃透明质感
- 椭圆导航和按钮
- 大留白
- 少装饰，多层次
- 中文正文阅读体验优先

### 6. 学习记录

每完成一个组件，写一段解释：

- 组件负责什么
- 数据从哪里来
- Astro 在构建时做了什么
- CSS 为什么这样写
- 以后想改应该改哪里

## v0.2 完成标准

- 可以只写 Markdown 就新增文章。
- 首页自动显示最新文章。
- `/posts/文章-slug/` 自动生成文章页。
