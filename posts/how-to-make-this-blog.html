如何制作这样的博客

第一版不要追求复杂系统。先把写作、页面、部署和版本记录跑通。

中文草稿

这个博客的第一版选择零构建静态页面。原因很简单：现在最重要的不是框架能力， 而是形成一个可以持续更新的写作和发布流程。

内容先用 Markdown 起草，页面先用 HTML 和 CSS 呈现。等文章数量变多，再决定是否迁移到 GitHub Pages 内置 Jekyll、Astro 或其他系统。这样每一步都能被解释，也能被版本管理。

设计上先固定信息架构和语气：科研、生活、作品集分区；首页保留一句硬核科幻感的宣言； 视觉上围绕“珲”这个字做书法字体、头像和组件系统。

English Draft

The first version uses plain static pages. The goal is not to win a framework debate. The goal is to establish a writing, publishing, and versioning loop.

Markdown is used as the source format, while HTML and CSS provide the visible prototype. When the archive grows, the site can migrate to Jekyll, Astro, or another system with less guesswork.

The design starts with structure and tone: research, life, and works. The home page carries one hard sci-fi sentence, and the identity grows from the calligraphic character “珲”.
